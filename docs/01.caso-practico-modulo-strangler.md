# EJERCICIO SDD: Módulo de Archivos Adjuntos

## Datos del Ejercicio
**Duración:** 2 horas
**Modalidad:** Parejas
**Dificultad:** Intermedia

---

## Objetivos de Aprendizaje

- Aplicar metodología Spec-Driven Development (SDD) a proyecto legacy
- Integrar nuevo módulo sin modificar código existente
- Generar artifacts SDD usando GitHub Copilot
- Practicar toma de decisiones técnicas con arquitectura Clean Architecture

## Caso Práctico: Requerimiento Recibido

**De:** Laura Martínez - Product Owner
**Asunto:** URGENTE: Adjuntos en tareas - Nuevo requerimiento

> Hola equipo,
>
> Necesitamos agregar la funcionalidad de **adjuntar archivos a las tareas**.
>
> **Requisitos funcionales:**
> 1. Los usuarios deben poder adjuntar archivos a cualquier tarea activa
> 2. Tipos permitidos: imágenes (jpg, png, gif), PDFs, documentos Office (doc, docx, xls, xlsx)
> 3. Tamaño máximo por archivo: 10 MB
> 4. Máximo 5 archivos adjuntos por tarea
> 5. Los usuarios deben poder descargar los archivos subidos
> 6. Los archivos se eliminan cuando se elimina la tarea asociada

>
> **Restricciones:**
> - Mantener arquitectura legacy intacta
> - No modificar entidades existentes (Task, User, Project)
> - Usar autenticación JWT existente
> - Integrar con endpoints de tareas
>
> Saludos,
> **Laura Martínez**
